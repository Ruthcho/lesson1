
כאשר מתבצעת התאמה מדויקת בין הכתובת והמתודה שהגיעו מהלקוח לבין מה שהגדרת ב-Express, הקוד מפעיל פונקציה מיוחדת (הנקראת Handler או Callback) שמטפלת בבקשה ומחזירה תשובה.
שלוש דוגמאות של נתיבי GET ב-Express
הנה שלוש דוגמאות נפוצות לנתיבים המשתמשים ב-app.get(), המציגות סוגים שונים של תגובות שהשרת יכול להחזיר:
דוגמה 1: נתיב השורש הסטטי (Root Route)
נתיב בסיסי שמחזיר לרוב את נקודת ההתחלה של האתר או הודעת מערכת פשוטה.
JavaScript
app.get('/', (req, res) => {
    res.send('<h1>ברוכים הבאים לשרת ה-Express שלי!</h1>');
});
•	מה הנתיב הזה עושה? הוא מאזין לכתובת הראשית של האתר (למשל http://localhost:3000/).
•	מה הוא מחזיר? הוא מחזיר קוד HTML פשוט (במקרה זה, כותרת <h1>). הדפדפן ירנדר את ה-HTML הזה ויציג למשתמש טקסט מעוצב על המסך.
דוגמה 2: נתיב לקבלת אוסף נתונים (Collection Route)
נתיב API קלאסי המשמש לשליפת רשימה מלאה של פריטים מתוך מאגר הנתונים.
JavaScript
app.get('/api/products', (req, res) => {
    const productsList = [
        { id: 1, name: 'מחשב נייד', price: 3500 },
        { id: 2, name: 'עכבר אלחוטי', price: 150 }
    ];
    res.json(productsList);
});
•	מה הנתיב הזה עושה? הוא מאזין לבקשות המגיעות לכתובת /api/products.
•	מה הוא מחזיר? הוא מחזיר מערך של אובייקטים בפורמט JSON. קוד ה-JavaScript בצד הלקוח (ה-Frontend) יקבל את הנתונים הגולמיים הללו, וישתמש בהם כדי לבנות דינמית כרטיסיות מוצר עבור חנות מקוונת.
דוגמה 3: נתיב דינמי עם פרמטר (Parametric Route)
כפי שראינו קודם, נתיב המשתמש בסימן הנקודתיים (:) כדי לקבל משתנה דינמי מהכתובת ולשלוף פריט אחד ספציפי.
JavaScript
app.get('/api/products/:productId', (req, res) => {
    const idRequested = req.params.productId;
    
    // כאן השרת יחפש בבסיס הנתונים רק את המוצר שה-ID שלו שווה ל-idRequested
    res.json({
        id: idRequested,
        name: 'מקלדת מכנית',
        status: 'במלאי'
    });
});
•	מה הנתיב הזה עושה? הוא מאזין לכל כתובת שתתחיל ב-/api/products/ ואחריה מספר או מזהה כלשהו (למשל /api/products/99).
•	מה הוא מחזיר? הוא מחזיר אובייקט JSON בודד המכיל אך ורק את הפרטים המלאים של אותו המוצר הספציפי שנתבקש, ולא את כל רשימת המוצרים.
על פי התמונה החמישית שהעלית (image_8db440.png), המשימה כעת היא להסביר מהו Module (מודול) ב-Node.js, להסביר כיצד קובץ JavaScript אחד יכול להשתמש בקוד מקובץ JavaScript אחר, ולהציג דוגמה קטנה המשתמשת ב-module.exports וב-require.
להלן ההסבר המלא והממוקד בעברית:
מה זה Module (מודול) ב-Node.js?
בצד הלקוח (Frontend) המסורתי, לעיתים קרובות טוענים מספר קבצי JavaScript באמצעות תגיות <script> ב-HTML, וכל המשתנים הופכים לגלובליים ומעורבבים זה בזה.
ב-Node.js הגישה שונה לחלוטין: כל קובץ JavaScript הוא מודול מבודד בפני עצמו. משתנים, פונקציות או מחלקות (Classes) שאתה מגדיר בקובץ מסוים נשארים "פרטיים" ונעולים בתוך אותו קובץ בלבד. הם אינם זמינים לקבצים אחרים בפרויקט, אלא אם כן תבחר לייצא אותם במפורש.
מערכת המודולים הזו עוזרת לשמור על קוד מאורגן, מונעת התנגשויות בין שמות של משתנים, ומאפשרת לחלק פרויקטים גדולים לחלקים קטנים וקריאים (רכיבים עצמאיים).
כיצד קובץ אחד משתמש בקוד של קובץ אחר?
כדי להעביר קוד מקובץ א' לקובץ ב', אנו משתמשים בשני מנגנונים משלימים (בשיטת CommonJS הנפוצה ב-Node.js):
1.	module.exports (היצוא): בקובץ שבו כתבנו את הלוגיקה, אנו משתמשים באובייקט המובנה module.exports כדי להגדיר מה מתוך הקובץ יהיה חשוף החוצה לעולם.
2.	require() (היבוא): בקובץ שבו אנו רוצים להשתמש בלוגיקה הזו, אנו קוראים לפונקציה המובנית require() ומספקים לה את נתיב הקובץ שממנו אנו רוצים לייבא.
דוגמה מעשית קטנה
נניח שיש לנו שני קבצים באותה התיקייה: קובץ עזר שמכיל פונקציה מתמטית, וקובץ השרת הראשי שרוצה להשתמש בה.
שלב 1: יצירת הקובץ המייצא (mathUtils.js)
כאן נגדיר פונקציה פשוטה ונבחר לייצא אותה:
JavaScript
// פונקציה פרטית בתוך המודול
function add(a, b) {
    return a + b;
}

// חשיפת הפונקציה החוצה כדי שקבצים אחרים יוכלו להשתמש בה
module.exports = add;
שלב 2: יצירת הקובץ המייבא ושימוש בקוד (app.js)
כאן ניבוא את הפונקציה בעזרת require ונפעיל אותה:
JavaScript
// ייבוא הפונקציה מהקובץ השכן (חובה להשתמש ב- ./ כדי לציין שמדובר בקובץ מקומי)
const addFunction = require('./mathUtils');

// שימוש בפונקציה המיובאת
const result = addFunction(5, 10);

console.log(result); // יידפס: 15
נקודה חשובה:
שים לב שבקובץ המייבא (app.js) יכולנו לקרוא למשתנה בשם addFunction (או כל שם אחר שנרצה), מכיוון ש-require פשוט מחזירה את מה שקובץ המקור שם בתוך module.exports
